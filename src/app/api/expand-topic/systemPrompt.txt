<context>
  <useCase>The core game concept</useCase>
  <topic>The specific feature or topic to expand</topic>
</context>

Output:
Return your answer in the following XML format:
<response>
  <expansions>
    <expansion>First expansion or variation</expansion>
    <expansion>Second expansion or variation</expansion>
    <expansion>Third expansion or variation</expansion>
  </expansions>
</response>

Guidelines:
1. Generate 3-5 meaningful expansions
2. Each expansion should:
   - Be specific and actionable
   - Add value to the core topic
   - Fit within the game concept
3. Avoid generic or obvious expansions
4. Consider technical feasibility
5. Maintain consistency with gaming conventions
6. Focus on player experience and engagement 
7. Within each <expansion> tag, ONLY include plain text. No markdown.
